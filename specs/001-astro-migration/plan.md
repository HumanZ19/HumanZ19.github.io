## Summary

Migrate the existing legacy static HTML portfolio at `https://github.com/HumanZ19/HumanZ19.github.io` to a production-ready Astro 4.x architecture. The new site is data-driven (single JSON source of truth), type-safe (TypeScript strict + Zod), bilingual (FR/EN with Astro i18n), and targets Lighthouse ≥95. All 20 projects and complete profile data extracted from the existing repo are migrated into structured JSON. The site deploys to GitHub Pages via GitHub Actions.

### Dark Mode Init Script (blocking, inline in BaseLayout)

```javascript
// Must run before first paint — inline in <head>
const stored = localStorage.getItem('theme');
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
if (stored === 'dark' || (!stored && prefersDark)) {
  document.documentElement.classList.add('dark');
}
```

## Complexity Tracking

| Item | Justification | Alternative Rejected |
|------|--------------|---------------------|
| Puppeteer in CI | PDF generation from HTML print page is the most reliable cross-platform method | Manual PDF upload requires Karim to regenerate on every content change |
| Astro islands for 4 features | Dark mode needs blocking script; filters/carousel need JS for UX; all other content is static | SSR would add complexity and break static GitHub Pages deployment |
| Zod validation at build time | Catches data errors before deployment rather than at runtime | TypeScript interfaces alone don't validate JSON at runtime |
